# 系统管理工作手册

## Description
提供系统管理相关操作的标准化流程，包括系统信息查询和监控、系统配置和优化、服务管理和启动/停止、系统资源管理和系统安全检查等任务。

## Input
用户的系统管理请求，包含以下信息：
- 操作类型：系统信息查询、服务管理、资源监控、系统配置或安全检查
- 目标对象：操作的目标系统或服务
- 操作参数：根据具体操作类型提供的参数
- 操作指令：具体的系统命令或操作步骤

## Output
- 执行结果：操作是否成功完成
- 系统信息：查询到的系统信息或监控数据
- 详细报告：执行过程和结果的详细信息
- 错误信息：如果操作失败，提供错误原因

## Steps
1. **分析用户请求**：理解用户想要执行什么系统操作
2. **验证操作权限**：检查是否有执行该操作的权限
3. **制定执行计划**：根据任务类型选择合适的系统工具和命令
4. **执行系统操作**：运行系统命令或使用系统工具执行操作
5. **验证执行结果**：检查操作是否成功完成
6. **处理响应数据**：解析和处理命令输出，提取有用信息
7. **生成执行报告**：向用户提供详细的执行结果和系统状态

## Examples
### 示例1：查询系统信息
**输入**：获取当前系统的详细信息
**步骤**：
1. 分析用户请求，确定需要获取的系统信息类型
2. 构建系统信息查询命令
3. 执行系统信息命令
4. 解析命令输出，提取关键信息
5. 整理和格式化信息
6. 生成系统信息报告

### 示例2：服务管理
**输入**：重启Apache服务
**步骤**：
1. 分析用户请求，确定服务名称和操作类型
2. 构建服务重启命令
3. 执行服务重启命令
4. 验证服务是否成功重启
5. 生成服务管理报告

## Notes
- **权限检查**：确保有执行系统命令的权限
- **安全考虑**：避免执行可能损害系统的命令
- **错误处理**：妥善处理命令执行错误和异常情况
- **输出处理**：正确处理命令输出，提取有用信息
- **系统兼容性**：考虑不同系统版本的兼容性

## Tools
- ShellTool：执行系统Shell命令
- SystemTool：系统信息查询和管理
- NetworkTool：网络相关的系统操作

## System Operations
### 系统信息查询
- **硬件信息**：CPU、内存、磁盘、网络适配器等
- **软件信息**：操作系统版本、内核版本、已安装软件等
- **网络信息**：IP地址、网络连接、网络状态等

### 服务管理
- **启动服务**：启动系统服务或应用程序
- **停止服务**：停止运行中的服务
- **重启服务**：重启服务以应用配置更改
- **服务状态**：检查服务的运行状态

### 资源监控
- **CPU使用率**：监控CPU的使用情况
- **内存使用**：监控内存使用情况和交换空间
- **磁盘空间**：检查磁盘分区的使用情况
- **网络流量**：监控网络接口的流量情况

### 系统维护
- **磁盘清理**：清理临时文件和垃圾文件
- **系统更新**：检查和安装系统更新
- **日志查看**：查看系统日志和应用程序日志
- **备份恢复**：执行系统备份和恢复操作
