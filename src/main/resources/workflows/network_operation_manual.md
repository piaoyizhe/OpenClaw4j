# 网络操作工作手册

## Description
提供网络相关操作的标准化流程，包括HTTP/HTTPS请求发送、API调用、网络服务状态检查、资源下载上传和网络连接测试等任务。

## Input
用户的网络操作请求，包含以下信息：
- 操作类型：API调用、网络测试、资源下载、网络监控或数据上传
- 目标地址：API URL或网络服务地址
- 请求参数：根据具体操作类型提供的参数
- 请求方法：GET、POST、PUT、DELETE等（如果适用）
- 认证信息：API密钥、令牌等（如果需要）

## Output
- 执行结果：操作是否成功完成
- 响应数据：API返回的数据或网络测试结果
- 详细报告：执行过程和结果的详细信息
- 错误信息：如果操作失败，提供错误原因

## Steps
1. **分析用户请求**：理解用户想要执行什么网络操作，目标是什么
2. **验证网络参数**：检查URL、请求方法、参数等是否正确
3. **制定执行计划**：根据任务类型选择合适的网络工具和参数
4. **执行网络操作**：发送请求并处理响应
5. **验证执行结果**：检查操作是否成功，响应数据是否正确
6. **处理响应数据**：解析和处理响应数据，提取有用信息
7. **生成执行报告**：向用户提供详细的执行结果和数据

## Examples
### 示例1：调用天气API获取城市天气信息
**输入**：获取北京市的天气信息
**步骤**：
1. 构建天气API的请求URL
2. 设置请求参数（城市名称、API密钥等）
3. 发送HTTP请求
4. 解析JSON响应
5. 提取天气信息
6. 生成执行报告

### 示例2：网络连接测试
**输入**：测试与google.com的网络连接
**步骤**：
1. 构建测试命令
2. 执行网络连接测试
3. 分析测试结果
4. 生成测试报告

## Notes
- **URL验证**：确保URL格式正确，避免恶意URL
- **请求参数**：检查请求参数是否完整和正确
- **超时设置**：合理设置请求超时时间，避免长时间阻塞
- **错误处理**：妥善处理网络错误和异常情况
- **数据安全**：注意保护敏感数据，避免泄露
- **响应解析**：正确解析不同格式的响应数据

## Tools
- HttpTool：执行HTTP/HTTPS请求
- ShellTool：执行网络相关的Shell命令
- NetworkTool：网络连接测试和监控

## HTTP Methods
- **GET**：获取资源，适合查询操作
- **POST**：提交数据，适合创建或更新操作
- **PUT**：更新资源，适合完整更新操作
- **DELETE**：删除资源
- **PATCH**：部分更新资源

## Status Codes
- **200 OK**：请求成功
- **201 Created**：资源创建成功
- **400 Bad Request**：请求参数错误
- **401 Unauthorized**：未授权
- **403 Forbidden**：禁止访问
- **404 Not Found**：资源不存在
- **500 Internal Server Error**：服务器内部错误
- **503 Service Unavailable**：服务不可用
