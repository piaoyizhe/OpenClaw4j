# 文件管理工作手册

## Description
提供文件系统操作的标准化流程，包括文件整理、查询、操作、目录管理和文件统计等任务。

## Input
用户的文件管理请求，包含以下信息：
- 操作类型：文件整理、查询、操作、目录管理或文件统计
- 目标文件/目录：操作的目标文件或目录路径
- 操作参数：根据具体操作类型提供的参数

## Output
- 执行结果：操作是否成功完成
- 详细报告：执行过程和结果的详细信息
- 统计数据：如果是统计操作，提供相关统计信息

## Steps
1. **分析用户请求**：理解用户想要对哪些文件执行什么操作
2. **检查目标位置**：验证目标目录是否存在，权限是否正确
3. **制定执行计划**：根据任务类型和文件特性制定详细的执行计划
4. **查询系统信息**：使用get_system_info判断是什么系统windows linux ubuntu 还是 macOS
5. **执行文件操作**：根据不同系统 使用Shell命令或文件工具执行具体操作
6. **验证执行结果**：检查操作是否成功完成，文件是否正确处理
7. **生成执行报告**：向用户提供详细的执行结果和统计信息
8. **发送邮件**：如果用户需要发送邮件可以使用 send_email


## Notes
- **保留现有结构**：不操作现有的文件夹，只处理零散文件
- **文件权限**：注意文件权限和安全问题
- **大文件处理**：处理大文件时注意性能和超时问题
- **错误处理**：妥善处理执行过程中可能出现的错误
- **用户反馈**：提供清晰、详细的执行结果报告

## Tools
- get_system_info：获取系统信息，包括操作系统、Java版本等
- send_email：发送邮件到指定邮箱
- execute_shell：用于使用命令操作文件
